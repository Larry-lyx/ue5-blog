Damage Text

发表于 2025-12-19 分类于 SystemDesigns ， UI

核心总结

伤害提示（Damage Text）是提升战斗体验的核心UI效果，通过Widget（视觉展示）、Widget Component（初始位置锚定）、伤害结算逻辑（触发时机）三层配合实现；可根据伤害类型（暴击Critical Hit、格挡Blocked Hit）定制视觉表现（颜色、动画），并通过PlayerController统一管控Widget的生成与销毁，既保证提示位置精准，又避免组件冗余占用内存。

伤害提示的实现逻辑

战斗中需实时显示攻击伤害数值，并通过视觉差异化区分特殊攻击效果，核心依赖三个模块：Widget（最终视觉输出）、Widget Component（初始位置锚定）、伤害结算逻辑（触发时机）。

1. 触发时机（伤害结算后）

伤害提示需在伤害数值最终确定后触发，因此核心触发逻辑放在AttributeSet（AS）的伤害处理函数中（调用自InComingDamage等元属性处理逻辑）；但UI相关操作需由PlayerController（PC）统一管控，因此AS中完成数值校验后，将逻辑转发至PC执行：

核心设计逻辑

优先通过攻击者（SourceCharacter）的PC触发，攻击者未持有PC时（如AI），再通过被攻击者（TargetCharacter）的PC兜底，确保伤害提示稳定触发。

2. Widget Component（初始位置锚定）

Widget Component仅用于锚定伤害提示的初始生成位置（敌人RootComponent周围），生成后立即解绑以支持自定义运动轨迹；PC中完成Component的创建、锚定、参数传递，并最终销毁组件避免内存泄漏：

关键步骤说明

  • 注册组件：NewObject创建后需调用RegisterComponent确保组件生效；
  • 锚定+解绑：先Attach到敌人根组件确定初始位置，再Detach解绑，后续可自由控制运动；
  • 参数传递：通过SetDamageText将伤害数值、攻击类型标记传递给Widget；
  • 内存管控：组件完成提示展示后需销毁，避免大量无效组件堆积。

3. Widget（视觉表现层）

Widget是伤害提示的最终展示载体，核心负责动画效果和内容定制：

3.1 动画设计

制作差异化的动态效果（如放大高亮、向上飘移、渐隐消失等），在Widget的Construct阶段启动动画，保证伤害数字出现时有醒目且流畅的动态反馈。

3.2 内容定制

根据传入的bBlockedHit（格挡）、bCriticalHit（暴击）标记，动态调整文本的颜色、字体样式，并显示精准的伤害数值。


时序说明

Widget的Construct阶段会立即启动动画，而SetDamageText中需将Widget转换为WBP_DamageText并调用UpdateDamageText更新显示内容；因动画播放是异步持续过程，数据更新（数值、样式）执行速度远快于动画，因此视觉上能看到正确的伤害信息伴随动画完整展示，无时序错乱问题。
